操作系统的发展过程

发表于 2021-05-17 更新于 2021-07-01 分类于 操作系统
本文字数： 1.2k

操作系统主要分为：单道批处理系统、多道批处理系统、分时系统、实时系统

单道批处理系统

概念：系统对作业的处理都是成批进行的、且内存中始终只保持一道作业，称为单道批处理系统。

主要特征：
（1）自动性：一批作业自动逐个运行，无须人工干预
（2）顺序性：正常情况下作业按顺序完成
（3）单道性：内存中只有一道程序

缺点：
（1）CPU 利用率低
（2）内存浪费

多道批处理系统

多道：

多道的概念：系统中同时驻留多个作业，并使它们交替地运行，保持 CPU 处于忙碌状态。

多道的优点：
（1）提高 CPU 利用率
（2）提高内存和 I/O 设备利用率
（3）提高了系统吞吐量

多道批处理系统：

主要特征：
（1）多道性：内存中同时驻留多个程序，并发执行
（2）无序性：多个作业完成的先后顺序与它们进入内存顺序无对应关系
（3）调度性：完成两次调度 作业调度：从外存中选择若干个作业调入内存 进程调度：分配 CPU

优点：资源利用率高；系统吞吐量大；

缺点：平均周转时间长；无交互能力；

多道批处理系统需解决的问题：

  • 处理机管理问题
  • 内存管理问题
  • I/O管理问题
  • 文件管理问题
  • 作业管理问题
  • 接口管理问题

分时系统

概念：分时系统指一台主机上连接多个带有显示器和键盘的终端，同时允许多个用户共享主机中的资源，各个用户都可通过自己的终端以交互方式使用计算机。

主要特征：
（1）多路性：系统允许将多台终端同时连接到一台主机上
（2）独立性：多台终端之间互不干扰
（3）及时性：用户的请求能在很短的时间内获得响应
（4）交互性：用户可通过终端与系统进行广泛的人机对话

实时系统

概念：实时系统(Real-Time System)是指系统能及时(或即时)响应外部事件的请求，在规定的时间内完成对该事件的处理，并控制所有实时任务协调一致地运行。

分类：
（1）周期性实时任务：外部设备周期性地发出激励信号给计算机，要求它按指定周期循环执行，以便周期性地控制某外部设备。
（2）非周期性实时任务： 开始截止时间：任务在某时间前必须开始执行。 完成截止时间：任务在某时间前必须完成。
（3）硬实时任务：系统必须满足任务对截止时间的要求。
（4）软实时任务：系统如果偶尔错过了截止时间，产生的影响不大。

主要特征： （1）多路性 （2）独立性 （3）及时性 （4）交互性 （5）可靠性

从系统特征对比 分时系统 实时系统
多路性 按分时原则为多个终端用户服务 系统周期性地对多路现场信息进行采集，以及对多个对象或执行机构进行控制
独立性 每个终端用户彼此独立，互不干扰 对信息的采集和对对象的控制也是彼此互不干扰
及时性 用户角度：以用户所能接受的等待时间确定 程序角度：以控制对象所要求的截止时间确定
交互性 分时系统一般都是通用性很强的系统，允许系统和用户之间有较强的会话能力，交互性强。 实时系统大都是特殊的专用系统，外界操作是严格控制的，因此交互性弱。
可靠性 要求系统可靠 要求系统高度可靠
