Git 是一个免费的、开源的分布式版本控制系统

Git 概述

Git 是一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的各种项目。

像 Git 这种分布式版本控制工具，客户端提取的不是最新版本的文件快照，而是把代码仓库完整地镜像下来（本地库）。这样任何一处协同工作用的文件发生故障，事后都可以用其他客户端的本地仓库进行恢复。因为每个客户端的每一次文件提取操作，实际上都是一次对整个文件仓库的完整备份。

Git 有三种代码托管中心（基于网络服务器的远程代码仓库），一般简称为远程库：基于局域网的 GitLab、基于互联网的 GitHub、Gitee。

Git 常用命令

设置用户签名

签名的作用是为了区分不同操作者的身份，在每一个提交信息中都能够看到每次提交是由哪个用户发起的。

Git 首次安装必须设置一下用户签名，否则无法提交代码。

初始化本地仓库

在需要进行 Git 管理的文件夹中，执行该 Git 命令，会生成一个隐藏的 .git 文件，代表本次初始化完成。

.git 文件中包含了 HEAD（记录当前指向的分支）、以分支名命名的文件（记录了当前分支指向的版本号）等信息。

查看本地库状态

用于查看当前文件夹中的文件状态，如果有文件尚未添加到暂存区，则文件名为红色；如果已添加到暂存区而尚未提交到本地库，则是绿色；没有上述状态文件，则不显示。

添加暂存区

将文件添加到暂存区，但还未提交到本地库，不会生成一个历史版本。

提交本地库

将暂存区中的文件提交到本地库，此时会生成一个对应的历史版本，并记录了版本的提交者。

查看历史版本

可以看到历代版本的版本号、日志信息、提交者等。

版本穿梭

可以根据版本号切换版本，同时本地的文件也会切换到对应的版本。

版本的切换就是通过改变 master 指针来实现的。

Git 分支操作

概念

在企业中，一个产品一般分为三个运行环境：dev(开发)、test(测试)、prod(生产) 。

采用三个环境就是为了避免开发、测试、生产环境中的问题互相影响，在开发中即使出现大的 bug 也不会影响到已经上线的产品正常运作。

分支也是类似的，一般情况下，我们不会直接在 master 分支上进行修改，而是为每个任务创建单独分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来， 开发自己分支的时候，不会影响主线分支的运行。

创建分支

切换分支

切换到别的分支后，在此分支上对文件的修改不会影响到其他的分支。

分支的切换就是 HEAD 指针指向的改变。

合并分支

如果发生合并冲突，会在当前分支名的后面显示 MERGING）。

冲突产生的原因：合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。 Git 无法替我们决定使用哪一个。必须人为决定新代码内容。

冲突的解决：

  1. 使用 git status 查看冲突发生的位置。

  2. 打开发生冲突的文件，针对冲突的内容进行修改。

  1. 添加到暂存区 git add filename
  2. 执行提交 git commit -m "log"，此时使用命令时不能带具体的文件名。
  3. MERGING 状态消失，冲突解决。
