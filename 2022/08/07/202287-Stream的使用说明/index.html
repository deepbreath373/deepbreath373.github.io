Stream的使用说明

发表于 2022-08-07 更新于 2022-08-14 分类于 Java
本文字数： 6k

Stream 的翻译是流，是 Java 8 的一个新特性，可以用于对集合、数组等数据结构进行计算。

如何生成流？

  • 通过集合生成

  • 通过数组生成

    通过 Arrays.stream 方法生成的流是数值流IntStream，可以避免计算过程的拆装箱。

    Stream API 提供了 mapToInt、mapToDouble、mapToLong 三种方式将对象流Stream，转换成对应的数值流。

  • 通过值产生

  • 通过文件生成

    通过 Files.line 方法得到一个流，并且得到的每个流是给定文件中的一行。

  • 通过函数生成

    • iterate

      iterate 方法接收两个参数，一个为初始化值，一个是要进行的函数操作。因为 iterate 生成的流是无限的，所以使用 limit 的方法对返回结果进行截断。

    • generate

      generate 方法接受一个 Supplier 类型的参数，为流提供值。同时生成的流也是无限流，所以通过 limit 进行截断。

流的操作类型

中间操作

一个流可以后面跟随零个或多个中间操作，用来对数据进行加工，并将加功后的数据传递给下一个操作。类似于工厂的流水线，“数据”经过一个个“中间操作”加工、传递。这类操作都是惰性的，仅仅调用这些方法，并没有真正开始流的遍历，真正的遍历需要等到终端操作时。

filter 筛选

通过使用 filter 方法进行条件筛选，filter 方法的参数作为一个判断条件，保留条件判断为 true 的元素并返回。

distinct 去重

通过 distinct 方法快速去除重复的元素。

limit 返回指定流个数

通过 limit 限定返回流的个数，limit >= 0 否则抛出异常。

skip 跳过流中的元素

通过 skip 可以跳过流中的前几个元素，skip >= 0 否则抛出异常。

map 流映射

将接收的元素映射成另外一个元素，例如：String -> Integer

flatMap 流转换

将一个流中的每一个值都转换成另一个流。

map(w -> w.split(“ “)) 的返回值为 Stream<String[]>，想获取 Stream，可以通过flatMap方法完成 Stream ->Stream 的转换。

allMatch/anyMatch/noneMatch

  • allMatch 匹配所有元素

  • anyMatch 匹配任意一个元素

  • noneMatch 全部不匹配

终端操作

一个流有且仅有一个终端操作，当这个操作执行完成后，流就被关闭了。

count 统计流中元素个数

findFirst 查找第一个

findAny 随机查找一个

findAny 虽然是随机找一个符合条件的，但是也是当找到第一个满足的元素时就返回了。

提供 findAny 方法是为了更好的利用并行流，findFirst 方法在并行上限制更多。
